## Mechanism

**Advanced Encryption Standard (AES)** is a symmetric block cipher widely used for secure data encryption. It operates on **128-bit blocks** and supports key sizes of **128, 192, and 256 bits**.

Because AES only encrypts fixed-size blocks, a **mode of operation** is required to encrypt larger data such as shellcode. This project uses **AES Counter Mode (CTR)**, which effectively turns AES into a stream cipher.

In CTR mode, AES encrypts a continuously increasing counter value to generate a **keystream**, which is then XORed with the plaintext to produce ciphertext.

```
Plaintext XOR AES(Key, Counter) = Ciphertext
```

A key advantage of CTR mode is that the same operation is used for both encryption and decryption.

---

## 1. At Build Time (Python Builder)

During the build stage, the raw shellcode is encrypted using **Advanced Encryption Standard in **AES Counter Mode (CTR).**

The project uses tinyaes-py, a Python implementation compatible with tiny-AES-c, ensuring that encrypted payloads generated by the builder can be correctly decrypted by the C++ loader.

A random AES key and initialization vector (IV) are generated, and the shellcode is encrypted before being embedded into the loader.

Example:
```python
from tinyaes import AES
import os

key = os.urandom(32)   # AES-256 key
iv = os.urandom(16)

aes = AES(key)

ciphertext = aes.ctr_encrypt(shellcode, iv)
```

The following values are embedded into the loader:

- Encrypted payload
- AES key
- IV

These are exported as byte arrays and compiled into the C++ program.

## 2. At Runtime (C++ Loader)

Before execution, the loader decrypts the encrypted payload using tiny-AES-c, which shares the same algorithm implementation as the Python builder.

```python
AES_ctx ctx;

AES_init_ctx_iv(&ctx, aes_key, iv);

AES_CTR_xcrypt_buffer(&ctx, payload, payload_len);
```

Because CTR mode is symmetric, the same function performs both encryption and decryption.

## Analysis

### Strengths

**Stronger Obfuscation than XOR**

AES produces highly randomized ciphertext, making it significantly harder to identify recognizable byte patterns compared to simple XOR encoding.

**Widely Trusted Encryption Standard**

AES is the globally adopted symmetric encryption standard and is widely used in secure systems.

**Effective Against Static Signature Detection**

Encrypted payloads appear as high-entropy data, preventing straightforward signature matching against raw shellcode.

---

### Weaknesses

**Payload Exposure in Memory**

Like XOR, the payload must eventually be decrypted before execution. Once decrypted, the shellcode becomes visible to memory scanning techniques used by AV/EDR solutions.

**Embedded Key Risk**

If the encryption key is stored directly in the loader binary, it can be extracted through reverse engineering.

**Detectable Decryption Routine**

Security tools may identify cryptographic routines or suspicious memory transformations associated with runtime payload decryption.

---

## Summary

AES encryption provides a **stronger payload transformation layer than XOR**, effectively preventing static detection of embedded shellcode.

However, similar to other runtime decryption techniques, it **does not protect the payload once it is restored in memory**.

Therefore, AES should be considered **one layer in a multi-stage evasion strategy**, rather than a standalone defense against detection.
